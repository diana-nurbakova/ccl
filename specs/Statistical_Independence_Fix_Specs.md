# Statistical Independence Fix: Reanalysis Specifications

## Purpose

EDM Reviewer 2 identified that several analyses treat nested observations as independent. This document specifies the replacement analyses for each experiment. The goal is to either (a) run mixed-effects models that account for the nesting structure, or (b) demonstrate that existing analyses already operate at the correct level of aggregation. Both options are provided for each experiment.

---

## Overview of the Problem

| Experiment | Current test | Nesting structure | Reviewer's concern |
|---|---|---|---|
| A (FRANK) | Fisher's exact test on 3,249 flagged sentences | Sentences nested within summaries (4,942 sentences from ~2,470 summaries from 9 models) | Sentences from the same summary share content, model, and source article |
| A' (FELM) | Chi-squared on 4,426 segments | Segments nested within responses nested within prompts (847 prompts → responses → 4,426 segments) | Segments from the same response are dependent |
| B (Bastani) | Mann-Whitney U on 7,089 conversations | Conversations nested within students (~1,000 students × ~4 sessions) | Same student appears in multiple conversations |
| C (Bastani) | One-sample t-test on per-student slopes (N=451) | **Already at student level** — one slope per student | This analysis is already correct |
| D (WildChat) | One-sample t-test on per-user slopes (N=696) | **Already at user level** — one slope per user | This analysis is already correct |

**Experiments C and D are fine.** The unit of analysis is the student/user slope. No reanalysis needed. The t-tests operate on N=451 and N=696 independent observations respectively.

**Experiments A, A', and B need attention.** Details below.

## Experiment A (FRANK): Claim-Type Detection Asymmetry

### Current Analysis

- Unit: sentence (N=4,942 total; 3,249 flagged by ≥1 annotator)
- Test: Fisher's exact test on 2×2 table: {majority agreement vs. not} × {FACTUAL vs. INTERPRETIVE}
- Result: OR = 5.49, p < 10⁻¹¹⁶

### Nesting Structure

```
Model (9 models: BART, PEGASUS, etc.)
  └── Source article (CNN/DailyMail or XSum)
       └── Summary (one per model × article pair)
            └── Sentence (1–N per summary)
                 └── Annotation (3 per sentence)
```

The concern: sentences within the same summary share context (same source article, same model, same generation). If a model systematically produces more factual or interpretive errors, sentences are correlated within summary.

### Data Required

The raw FRANK data file: `data/human_annotations_sentence.json` from `https://github.com/artidoro/frank`

Each record contains:
- `hash`: unique sentence identifier
- `model_name`: which model generated the summary
- `article_id` or similar: which source article
- `summary_id` or equivalent: which summary the sentence belongs to
- Per-annotator error-type annotations (3 annotators × 7 error types)

### Option A: Mixed-Effects Logistic Regression (preferred)

**Model specification:**

For each flagged sentence $i$ in summary $j$ generated by model $k$:

```
majority_agreement_ij ~ ccl_category + (1 | summary_id) + (1 | model_name)
```

Where:
- `majority_agreement` = binary (1 if ≥2/3 annotators agree on the flagged error, 0 otherwise)
- `ccl_category` = FACTUAL vs. INTERPRETIVE (drop SOURCE for the primary comparison, or include as a 3-level factor)
- `(1 | summary_id)` = random intercept for summary (accounts for within-summary correlation)
- `(1 | model_name)` = random intercept for model (accounts for model-level variance)

**What to report:**
- Fixed effect of `ccl_category`: odds ratio + 95% CI + p-value
- Variance components for random effects
- Compare OR from GLMM to the current Fisher OR (5.49) — expect the direction to hold but the OR may change

**Implementation (Python):**

```python
import pandas as pd
import statsmodels.formula.api as smf
# or: from pymer4.models import Lmer  (if using R-style lme4 via pymer4)

# Load FRANK annotations
# Each row = one sentence, with columns for:
#   summary_id, model_name, ccl_category, majority_agreement (binary)

# Filter to flagged sentences only (where ≥1 annotator flagged an error)
# Filter to FACTUAL and INTERPRETIVE categories

model = smf.mixedlm(
    "majority_agreement ~ C(ccl_category, Treatment(reference='INTERPRETIVE'))",
    data=df_flagged,
    groups=df_flagged["summary_id"],
    # If statsmodels supports crossed random effects:
    # re_formula="1",
    # vc_formula={"model_name": "0 + C(model_name)"}
)
result = model.fit()
print(result.summary())
```

**Note on crossed random effects:** `statsmodels.mixedlm` supports only a single grouping factor. For crossed random effects (summary_id + model_name), use either:
- `pymer4` (Python wrapper for R's `lme4`): `Lmer("majority_agreement ~ ccl_category + (1|summary_id) + (1|model_name)", data=df, family="binomial")`
- R directly via `lme4::glmer`
- Or nest: since each summary comes from exactly one model, model_name is a higher-level grouping. Use `(1 | model_name/summary_id)` which is a nested random effect.

**Implementation (R):**

```r
library(lme4)

# df_flagged has columns: majority_agreement, ccl_category, summary_id, model_name
# majority_agreement is 0/1
# ccl_category is "FACTUAL" or "INTERPRETIVE"

model <- glmer(
  majority_agreement ~ ccl_category + (1 | model_name/summary_id),
  data = df_flagged,
  family = binomial
)
summary(model)
# Extract OR:
exp(fixef(model)["ccl_categoryFACTUAL"])
# 95% CI:
exp(confint(model, method = "Wald")["ccl_categoryFACTUAL", ])
```

### Option B: Summary-Level Aggregation (simpler, conservative)

Aggregate to summary level before testing:
1. For each summary, compute: proportion of flagged factual sentences with majority agreement, and proportion of flagged interpretive sentences with majority agreement
2. Run a paired Wilcoxon signed-rank test (or paired t-test) across summaries that have both factual and interpretive flagged sentences
3. Report the median difference and p-value

```python
# Per-summary agreement rates
summary_stats = df_flagged.groupby(['summary_id', 'ccl_category']).agg(
    agreement_rate=('majority_agreement', 'mean'),
    n_sentences=('majority_agreement', 'count')
).reset_index()

# Pivot to wide format: one row per summary, columns for factual_rate and interpretive_rate
wide = summary_stats.pivot(index='summary_id', columns='ccl_category', values='agreement_rate')
# Keep only summaries with both categories present
paired = wide.dropna(subset=['FACTUAL', 'INTERPRETIVE'])

from scipy.stats import wilcoxon
stat, p = wilcoxon(paired['FACTUAL'], paired['INTERPRETIVE'])
print(f"Wilcoxon: stat={stat}, p={p}")
print(f"Median factual rate: {paired['FACTUAL'].median():.3f}")
print(f"Median interpretive rate: {paired['INTERPRETIVE'].median():.3f}")
```

### Expected Outcome

The OR will almost certainly remain large and significant. The 2.4× gap in majority agreement (67.9% vs. 27.8%) is too large to be an artifact of within-summary correlation. The GLMM OR may be somewhat smaller than 5.49 (standard errors will be larger), but the direction and significance are expected to hold.

## Experiment A' (FELM): Claim-Type Error Rates in ChatGPT

### Current Analysis

- Unit: segment (N=4,426)
- Test: Chi-squared across 3 CCL categories; Fisher's exact for FACTUAL vs. INTERPRETIVE
- Result: χ² = 43.87, p < 0.0001; OR = 1.87

### Nesting Structure

```
Prompt (847 prompts)
  └── ChatGPT response (1 per prompt)
       └── Segment (1–N per response; mean ~5.2 segments/response)
```

Segments within the same response are dependent: they share the same prompt, the same generation context, and adjacent segments may be topically correlated. The FELM domain is constant within a response (a prompt belongs to one domain), so segments within a response always belong to the same CCL category.

### Key Insight: The Nesting is Between Groups, Not Within

Because all segments from a single prompt share the same domain (and therefore the same CCL category), **there is no within-prompt variation in the predictor variable**. A mixed-effects model with prompt as random intercept would have the CCL category perfectly confounded with the grouping factor.

This means the appropriate analysis is at the **prompt level**, not the segment level.

### Option A: Prompt-Level Aggregation (correct approach)

1. For each of the 847 prompts, compute: `error_rate` = (number of error segments) / (total segments)
2. Assign each prompt its CCL category based on domain
3. Compare error rates across CCL categories using a Kruskal-Wallis test (non-parametric) or one-way ANOVA

```python
# Aggregate to prompt level
prompt_stats = df_felm.groupby(['prompt_id', 'domain', 'ccl_category']).agg(
    n_segments=('is_error', 'count'),
    n_errors=('is_error', 'sum'),
).reset_index()
prompt_stats['error_rate'] = prompt_stats['n_errors'] / prompt_stats['n_segments']

# Compare across CCL categories
from scipy.stats import kruskal
groups = [g['error_rate'].values for _, g in prompt_stats.groupby('ccl_category')]
H, p = kruskal(*groups)
print(f"Kruskal-Wallis: H={H:.2f}, p={p:.6f}")

# Pairwise: FACTUAL vs INTERPRETIVE
from scipy.stats import mannwhitneyu
factual = prompt_stats[prompt_stats['ccl_category'] == 'FACTUAL']['error_rate']
interpretive = prompt_stats[prompt_stats['ccl_category'] == 'INTERPRETIVE']['error_rate']
U, p = mannwhitneyu(factual, interpretive, alternative='greater')
print(f"Mann-Whitney (FACTUAL > INTERPRETIVE): U={U}, p={p:.6f}")

# Effect size: rank-biserial correlation or median difference
print(f"Median error rate FACTUAL: {factual.median():.3f}")
print(f"Median error rate INTERPRETIVE: {interpretive.median():.3f}")
```

### Option B: GEE (Generalised Estimating Equations)

If you want to keep the segment level and account for within-prompt clustering:

```python
import statsmodels.api as sm
from statsmodels.genmod.generalized_estimating_equations import GEE
from statsmodels.genmod.families import Binomial
from statsmodels.genmod.cov_struct import Exchangeable

# But note: CCL category is constant within prompt, so the GEE
# is effectively comparing between-prompt means — same as Option A.
# GEE is the technically correct way to do this at segment level,
# but prompt-level aggregation is simpler and equally valid.
```

### Expected Outcome

The prompt-level analysis will have fewer observations (847 vs. 4,426) and therefore wider confidence intervals, but the effect direction should hold. The segment-level chi-squared inflates significance by treating segments as independent; the prompt-level test gives honest p-values.

### What to Report in the Paper

Replace: "χ² = 43.87, p < 0.0001; OR = 1.87"

With: "A prompt-level comparison confirms that ChatGPT produces factual errors at a higher rate than interpretive errors (median prompt-level error rate: X% vs. Y%; Mann-Whitney U = ..., p = ...). This result accounts for the within-response dependency structure."

Keep the segment-level rates (24.0% vs. 14.5%) as descriptive statistics, but attribute the inferential test to the prompt-level analysis.

## Experiment B (Bastani): Structure vs. Disposition

### Current Analysis

- Unit: conversation (N=7,089)
- Test: Mann-Whitney U on per-conversation metrics (turns, words/turn, evaluative rate, passive rate)
- Result: d = 0.99 for turns/conversation

### Nesting Structure

```
Class (school class — the Bastani RCT randomisation unit)
  └── Student (~1,000 students)
       └── Session (up to 4 per student)
            └── Conversation (1+ per session)
                 └── Turn (user turns within conversation)
```

The current analysis operates at the conversation level, which is good (not at the turn level). But conversations are nested within students (same student appears in up to 4 sessions), and students are nested within classes (the randomisation unit in Bastani's RCT).

### Data Required

From `https://github.com/obastani/GenAICanHarmLearning`:
- `text_analysis/data/raw/valid_student_data_w_time_stamp.csv` — conversation logs with student IDs, session IDs, timestamps
- `main_regressions/final_data.csv` — student-level data with class IDs and treatment assignment

You need to merge these to get: conversation-level metrics + student_id + class_id + treatment.

### Option A: Mixed-Effects Model (preferred)

**Model specification:**

For each conversation $i$ from student $j$ in class $k$:

```
metric_ij ~ treatment + (1 | student_id) + (1 | class_id)
```

Or nested: `treatment + (1 | class_id/student_id)`

Run separately for each metric (turns_per_conv, words_per_turn, evaluative_rate, passive_rate).

**Implementation (R):**

```r
library(lme4)

# For continuous metrics (turns_per_conv, words_per_turn):
model_turns <- lmer(
  turns_per_conv ~ treatment + (1 | class_id/student_id),
  data = df_conversations
)
summary(model_turns)

# For rate metrics (evaluative_rate, passive_rate — bounded [0,1]):
# Use beta regression or treat as continuous if rates aren't extreme
model_passive <- lmer(
  passive_rate ~ treatment + (1 | class_id/student_id),
  data = df_conversations
)
summary(model_passive)

# Cohen's d equivalent from mixed model:
# d = fixed_effect_estimate / sqrt(var_student + var_class + var_residual)
```

**Implementation (Python):**

```python
import statsmodels.formula.api as smf

# Option 1: statsmodels mixed model (single grouping factor)
model = smf.mixedlm(
    "turns_per_conv ~ treatment",
    data=df_conv,
    groups=df_conv["student_id"]
)
result = model.fit()
print(result.summary())

# Option 2: pymer4 for crossed/nested random effects
from pymer4.models import Lmer
model = Lmer("turns_per_conv ~ treatment + (1|class_id/student_id)", data=df_conv)
model.fit()
print(model.summary())
```

### Option B: Student-Level Aggregation (simpler)

1. Aggregate to student level: for each student, compute mean turns_per_conv, mean passive_rate, etc., across all their conversations
2. Run Mann-Whitney U at the student level (N ≈ 500 GPT Tutor + 500 GPT Base)
3. Compute Cohen's d at the student level

```python
# Aggregate to student level
student_means = df_conv.groupby(['student_id', 'treatment']).agg(
    mean_turns=('turns_per_conv', 'mean'),
    mean_passive=('passive_rate', 'mean'),
    mean_eval=('evaluative_rate', 'mean'),
    n_conversations=('turns_per_conv', 'count')
).reset_index()

from scipy.stats import mannwhitneyu
aug = student_means[student_means['treatment'] == 'aug']['mean_turns']
vanilla = student_means[student_means['treatment'] == 'vanilla']['mean_turns']
U, p = mannwhitneyu(aug, vanilla)

# Cohen's d at student level
d = (aug.mean() - vanilla.mean()) / np.sqrt((aug.std()**2 + vanilla.std()**2) / 2)
```

### Option C: Acknowledge + Argue (if time is short)

If rerunning is not feasible before May 8, acknowledge the nesting and argue conservativeness:

> "Per-conversation metrics are used as the unit of analysis. Conversations are nested within students (up to 4 sessions per student), which introduces dependence. To assess robustness, we also computed student-level aggregates (mean per-conversation metrics across sessions). The results are qualitatively identical: [report student-level d and p]. The published learning outcome analysis in Bastani et al. uses class-level cluster-robust standard errors and reports β = −0.064, p = 0.016 for GPT Base, confirming the direction of our conversation-level finding."

### Expected Outcome

The d = 0.99 for turns/conversation will almost certainly survive aggregation. It's a full standard deviation difference; within-student correlation cannot eliminate this. The d value may change somewhat (likely decrease slightly since student-level means will have less variance), but it will remain large. The evaluative rate difference (1% vs. 3%) is small in absolute terms and may lose significance at the student level — which would actually be fine, since our argument is that evaluative disposition *doesn't matter* (structure matters more).

## Experiments C and D: No Changes Needed

### Experiment C (Bastani trajectories)

The unit of analysis is already the per-student slope. Each student contributes one slope computed from their 3–4 sessions. The one-sample t-test on N=451 slopes is correct.

**One minor improvement** (optional): Report a mixed-effects model where the session-level metric is the DV and session number is the predictor, with student as random intercept. This is more powerful than the two-step approach (compute slopes, then test slopes) and accounts for unequal numbers of sessions.

```r
# Optional improvement for Exp C
```
